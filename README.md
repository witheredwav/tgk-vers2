# 🎧 Telegram Bot v3 — реферальная система со скидками + анти-фрод

## 🔴 Главный фикс: ModuleNotFoundError: No module named 'handlers'

Эта ошибка значит, что Railway запускает `bot.py` не из той директории, где лежит папка `handlers/`, либо папка `handlers/` не попала в репозиторий целиком (Git иногда не коммитит пустые `__init__.py` правильно, либо структура при загрузке "расплющилась").

**Проверь на GitHub репозитории:**

Открой репозиторий в браузере — там должна быть видна структура:
```
твой-репозиторий/
├── bot.py
├── config.py
├── database.py
├── keyboards.py
├── states.py
├── requirements.txt
└── handlers/
    ├── __init__.py
    ├── user.py
    ├── admin.py
    ├── broadcast.py
    └── referral.py
```

Если вместо папки `handlers/` ты видишь файлы `user.py`, `admin.py` и т.д. напрямую в корне (без папки) — это и есть причина ошибки. Нужно создать папку `handlers` в репозитории и положить туда эти 4 файла плюс `__init__.py`.

**Если структура верная, но ошибка всё равно есть** — на Railway зайди в Settings сервиса и проверь Root Directory: если там указан не корень репозитория, а вложенная папка, и физически `bot.py` лежит выше — поправь путь.

Также в `bot.py` я добавил защитный код, который сам подставляет правильную директорию в `sys.path` — это исправляет ситуацию, если Railway запускает процесс с другим working directory, но не поможет, если файлов `handlers/` физически нет в репозитории.

## 🎁 Реферальная система со скидками

### Как это работает для пользователя

1. В меню кнопка **«🤝 Реферальная программа»** показывает личную ссылку и прогресс по уровням
2. Друг переходит по ссылке → жмёт `/start` → бот запоминает, кто его пригласил
3. **Реферал засчитывается только когда друг подтвердит подписку на канал** (нажмёт «Проверить подписку» и реально окажется подписан) — это и есть защита от накрутки, см. ниже
4. При достижении порога пользователь получает уведомление и 9-значный код скидки

### Уровни (заданы в `config.py`, можно поменять)

```python
REFERRAL_TIERS = [
    (5, 10),   # 5 друзей → 10%
    (10, 15),  # 10 друзей → 15%
    (20, 20),  # 20 друзей → 20%
]
```

### Переход между уровнями

Когда пользователь поднимается с уровня 10% на 15% — старый код автоматически удаляется из базы, генерируется новый 9-значный код (буквы A-Z + цифры 0-9), отправляется новое уведомление. Старый код больше не существует и не может быть использован повторно.

### Получение скидки (вручную, как ты просил)

Пользователь присылает код тебе в личные сообщения → ты открываешь **админка → 🎁 Коды скидок** → находишь код → проверяешь кому он принадлежит и % скидки → после применения скидки нажимаешь 🗑 **удалить**, чтобы код больше не использовался повторно.

### Управление рефералами вручную

**Админка → 🏆 Рефералы** — список всех у кого есть приглашённые, с количеством. Нажав на пользователя — кнопки ➖1 / ➕1 / ➖5 / ➕5 для ручной корректировки (например, если заметил накрутку и хочешь снять рефералов, или хочешь добавить вручную за активность вне бота).

Важно: ручная корректировка не трогает лог подтверждённых рефералов (это честная история кто кого реально привёл), а только сдвигает итоговое число через отдельный множитель. Это значит можно безопасно опускать/поднимать счётчик без риска сломать связи между пользователями.

## 🛡 Защита от накрутки рефералов

Реализовано три уровня защиты:

**Подписка — не просто переход по ссылке.** Самая частая схема накрутки — открыть свою ссылку в 50 фейковых аккаунтах через `/start`. В этой реализации простой `/start` НЕ засчитывает реферала. Засчитывается только реальное прохождение проверки подписки на канал через Telegram API (`get_chat_member`). Создать 50 живых подписавшихся аккаунтов уже значительно дороже и медленнее, чем просто разослать ссылку.

**Защита от самоприглашения.** Если пользователь переходит по своей же ссылке (referrer_id == user_id), она игнорируется на уровне регистрации.

**Защита от повторного засчитывания.** Если пользователь уже был подтверждён как реферал ранее (например, вышел из подписки и зашёл снова), это не добавит второй "плюс" рефереру — список подтверждённых хранит уникальные ID.

### Что дополнительно можно сделать (не реализовано, но обсуждаемо)

Полностью исключить накрутку технически нельзя — человек с доступом к нескольким SIM-картам всё равно может создать несколько настоящих Telegram-аккаунтов и подписать их на канал. Если эта угроза реальна для тебя, следующий уровень защиты — ограничение по времени между регистрациями одного реферера (например, не более 1 нового подтверждённого реферала в час) или ручная проверка топа рефералов перед выдачей крупных скидок (благо кнопка для просмотра и корректировки уже есть в админке).
